# 6. 환율 데이터 모델

## 현재 문제

프로젝트 목표는
> 여러 환전소의 환율 비교

하지만 현재 데이터는

- Korea Eximbank API
- 내부 DB

만 존재하여 실제 환전소별 환율 비교가 어려움

---

## 해결 방안

### 1. 기준 환율

- Korea Eximbank API 사용
- 하루 1회 갱신

---

### 2. 환전소 우대율

- 환전소가 직접 입력
- 변경 시 즉시 반영

---

### 3. 사용자에게 보여줄 최종 환율

```
최종 환율
= 기준 환율 × (1 - 환전소 우대율)
```

---

### 4. 개발팀 확인 사항

- Eximbank API 호출 제한(Rate Limit)
- 실시간 갱신 가능 여부
- 환전소 우대율 관리용 관리자(Admin) 페이지 필요 여부
