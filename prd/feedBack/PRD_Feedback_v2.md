# PRD 2차 피드백 및 반영 내용

## 1. 환전소 등록 기능 (Admin)

### 피드백
- 환전소는 어디서 등록하는가?
- Admin에서 환전소를 등록할 수 있는 기능이 필요하다.

### 반영 내용
- Admin 페이지에 **환전소 등록 기능**을 추가한다.
- 관리자는 등록 폼을 통해 새로운 환전소를 등록 및 관리할 수 있다.
- 등록 가능한 정보는 다음과 같다.
  - 환전소명
  - 주소
  - 지도 위치(좌표)
  - 영업시간
  - 지원 통화
  - 연락처
  - 예약 가능 여부
  - 30분 단위 최대 예약 가능 인원(Time Slot Capacity)

---

## 2. 예약 확인 방식 변경 (QR Code)

### 피드백
- 예약 확인을 이미지로 할 경우 다른 사람이 이미지를 공유하여 사용할 수 있다.

### 반영 내용
- 예약 확인 방식을 **이미지에서 QR Code 방식으로 변경**한다.
- 예약 완료 시 예약별 고유 QR Code를 생성한다.
- 환전소 직원은 QR Code를 스캔하여 예약 정보를 확인하고 사용 처리한다.
- QR Code는 1회 사용을 원칙으로 하여 중복 사용을 방지한다.

---

## 3. 예약 시간 및 Waiting 정책

### 피드백
- 동일한 시간대에 예약이 몰리면 어떻게 처리하는가?
- Waiting 정책이 필요하다.

### 반영 내용
- 예약은 **30분 단위 Time Slot**으로 운영한다.
- 각 Time Slot마다 최대 예약 가능 인원을 설정한다.
- 최대 예약 가능 인원은 **환전소별로 다르게 설정**할 수 있다.
- 해당 값은 Admin이 직접 등록 및 수정할 수 있다.
- 최대 인원에 도달한 Time Slot은 예약이 불가능하도록 처리한다.

---

## 4. User Flow 및 Wireframe 보완

### 피드백
- Wireframe과 User Flow가 명확하지 않다.
- 환전소 예약 과정을 직관적으로 표현해야 한다.

### 반영 내용
예약 프로세스를 다음과 같이 명확하게 정의한다.

1. 지도에서 환전소 선택
2. 환전소 상세 정보 확인
3. 환율 및 예약 가능 시간 확인
4. 예약 정보 입력
5. 예약 완료
6. QR Code 발급
7. 예약 시간에 환전소 방문
8. QR Code 인증 후 환전 진행

---

## 5. 지도 표시 정책

### 피드백
- 지도에는 어떤 환전소가 표시되는가?

### 반영 내용
- 지도에는 **Admin이 등록한 환전소만 표시**한다.
- 등록되지 않은 환전소는 지도 및 검색 결과에 노출되지 않는다.
- 모든 지도 데이터는 Admin에서 관리하는 환전소 정보를 기반으로 제공한다.
