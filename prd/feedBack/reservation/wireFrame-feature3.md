## 2. PRD 대비 빠진 부분

### 2.1 KYC 1단계(휴대폰 본인인증) 화면이 없음
- PRD §7 사용자 플로우: "예약 정보 입력(방문 예정 일시) → **휴대폰 본인인증 완료(KYC 1단계)** → 예약 완료 알림 수신"
- PRD §14 API: `POST /auth/verify-phone` (KYC 1단계)가 별도 엔드포인트로 명시되어 있음
- → 현재 와이어프레임 흐름은 "시간 선택 → 예약 확인 → Confirm"으로 바로 이어지는데, **Confirm 이전(또는 예약 확인 화면 내)**에 휴대폰 인증번호 발송/입력 단계가 빠져 있음
- **반영 위치 제안**: RSV-02(예약 확인 화면) 이전에 `RSV-02-1 (Phone Verification)` 화면을 신설하거나, 예약 확인 화면 내 인증 컴포넌트로 삽입

### 2.2 2시간 홀드 / 만료 안내 문구 부재
- PRD §20.1 예약정책: 홀드는 "예약 시점으로부터 2시간", 만료 시 재고 자동 해제
- PRD §25 와이어프레임 요구사항(참고): Reservation 화면에 "**만료 안내 문구**" 필요 항목으로 명시되어 있음
- → 현재 예약 확인/Confirm 화면 설명에는 홀드 시간이나 만료 카운트다운 언급이 없음
- **반영 위치 제안**: 예약 확인 화면(Confirm 전 단계) 및 RSV-03(예약 조회 화면)에 "2시간 이내 방문하지 않으면 자동 취소됩니다" 형태의 문구 + 남은 시간 표시 추가

### 2.3 환율 고정(Rate Lock) 표시 누락
- PRD §8: 예약 처리 시 "환율 고정"이 명시적 처리 항목
- → 지금 화면 설명에는 예약 시점에 환율이 고정된다는 사실이 화면 요소로 드러나 있지 않음
- **반영 위치 제안**: 예약 확인 화면에 "적용 환율: OOO (예약 시점 고정, 변동 없음)" 문구 추가

### 2.4 고액 예약 제한(1만 달러) 대응 UI 없음
- PRD §20.1, §22.2: 미화 1만 달러 이상 예약은 차단, 기준액 근접 시 사전 안내 문구 필요
- → 현재 금액 입력/확인 화면에 이 제한에 대한 경고나 차단 처리가 없음
- **반영 위치 제안**: 금액 입력 화면(시간·통화 선택 화면과 통합되어 있다면 그 화면)에 한도 초과 시 경고 상태(disabled + 안내 문구) 추가

### 2.5 노쇼 이력 기반 예약 제한 얼럿 없음
- PRD §20.2: 최근 30일 노쇼 이력 있는 사용자는 동시 예약 1건 제한, 누적 3회 이상 시 7일 차단
- → 현재 정리된 엣지 케이스는 "시간대 마감" alert뿐이고, 노쇼 페널티로 인한 예약 차단 케이스는 없음
- **반영 위치 제안**: RSV-01 또는 시간·통화 선택 화면에 "예약 제한 상태"(신규 예약 불가 안내) 얼럿 케이스 추가

### 2.6 지점 선택(Branch 상세) 단계 불명확
- PRD §7 플로우: "환율 비교 리스트 확인 → **지점(환전소) 선택** → 예약 정보 입력"
- → 현재 RSV-01은 지도+AI추천 리스트만 있고, 핀/리스트 클릭 후 지점 상세(주소, 영업시간, 환율)를 보여주는 화면이 정리되어 있지 않음. 바로 시간 선택으로 넘어가는 구조인지 확인 필요
- **반영 위치 제안**: RSV-01과 시간 선택 화면 사이에 `RSV-01-1 (Branch Detail)` 화면 신설 검토, 혹은 지도 클릭 시 바텀시트 형태로 지점 정보 노출

### 2.7 통화/금액 입력 시점이 PRD 플로우와 어긋남
- PRD §7 플로우 순서: "로그인 → **통화 및 금액 입력** → 환율 비교 리스트 → 지점 선택 → 예약 정보 입력(일시)"
- → 즉 PRD에서는 통화/금액을 지점 선택 **이전**(Exchange Comparison 단계)에 입력하는 구조인데, 지금 와이어프레임 설명에는 "시간 선택 화면에서 통화를 선택해서 환율 진행"이라고 되어 있어 순서가 반대로 되어 있음
- **확인 필요**: 이 화면이 F2(환율비교) 플로우가 아니라 F3(Reservation) 안에서 통화를 다시 선택/확정하는 화면인지, 아니면 실제로 통화 입력 시점 자체가 재설계된 것인지 팀 내 확인 필요. 후자라면 PRD §7, §25 Exchange Comparison 설명도 함께 수정해야 함

### 2.8 미이행(노쇼 아닌 환전소 귀책) 대응 화면 없음
- PRD §21.4: 방문 시 예약 재고가 없는 경우(환전소 귀책) 즉시 알림 + 가장 가까운 대체 지점 자동 안내
- → 이 케이스는 방문 이후(현장) 상황이라 유저 앱보다는 알림/현장 안내 UI로 필요. 현재 F3 화면 정리에는 없음
- **반영 위치 제안**: 별도 알림(Notification) 또는 QR 화면(RSV-03-1)에서 "재고 없음, 대체 지점 안내" 상태 화면 추가 필요 여부 검토

### 2.9 예약 상태별 알림(Notification) 화면 부재
- PRD §6.1, §8: 예약 완료/취소/픽업 임박 알림이 기능 요구사항에 명시됨
- → 현재 F3 화면 목록에 알림 화면(또는 알림 진입 시 이동 경로)이 없음
- **반영 위치 제안**: 별도 플로우(F5 Notification 등)로 분리하거나, RSV-03 예약 조회 화면과 알림 연동 지점을 명시

## 3. 종합 제안 — RSV 화면 ID 재정리 (초안)

```
RSV-01     Reservation Home (지도 + AI 추천)
RSV-01-1   Branch Detail (지점 상세) ※ 신규 검토
RSV-02     Time & Currency Selection (30분 단위, 통화 선택) ※ 순서 확인 필요
RSV-02-1   Phone Verification (KYC 1단계) ※ 신규
RSV-02-2   Reservation Confirm (요약 + 환율고정 + 2시간 홀드 안내 + Confirm) ※ 항목 보강
RSV-03     Reservation Detail / QR·Cancel 진입
RSV-03-1   QR Code View (10초 갱신)
RSV-03-2   Cancellation
RSV-04     My Reservations (예약 리스트, PRD §25 항목: 상태 + 만료 카운트다운)
```
