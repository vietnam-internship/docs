# TravelX 와이어프레임 도메인 분류

와이어프레임은 CRUD 엔티티가 아니라 "사용자가 그 화면에 왜 들어가는지"를 기준으로 도메인을 나눈다. 사용자용 5개 도메인 + 관리자용 1개 도메인, 총 6개로 구성한다.

## 1. 인증 (Auth)

진입점 역할의 작고 단순한 도메인.

- 회원가입
- 로그인
- (선택) 비밀번호 찾기

## 2. 환율 정보 조회 (Rate Discovery)

예약 전에 "둘러보는" 성격의 화면들을 묶는다.

- 환율 홈 / 통화별 환율 리스트
- 통화 검색
- 지점별 환율 비교
- 환율 인사이트 (백분위 · 추세 · 변동성 등 참고 정보)

## 3. 환전 예약 프로세스 (Reservation Flow)

단계형 위저드(wizard) 흐름. 다른 도메인과 성격이 달라 반드시 독립시킨다. 각 스텝이 와이어프레임 한 장씩 나오는 구조.

1. 통화 · 금액 선택
2. 지점 선택
3. 픽업 일시 선택
4. 예약 확인 (요약)
5. 예약 완료 (QR 발급)

## 4. 예약 관리 (My Reservations)

"예약을 만드는 것"과 "만든 예약을 확인·관리하는 것"은 사용자 의도가 달라 프로세스 도메인과 분리한다.

- 예약 목록 (예정 / 완료 / 취소 필터)
- 예약 상세 (QR 재확인)
- 예약 취소
- 예약 이력

## 5. 마이페이지 & 알림 (My Page & Notification)

규모가 작아 하나로 묶어도 무방하다.

- 프로필 / 계정 정보
- 알림함 (예약 완료 · 취소 · 픽업 리마인드)

## 6. 관리자 (Admin) — 완전히 별도 세트

사용자 화면과 IA 자체가 다르므로 Figma 페이지도 처음부터 분리한다.

- 대시보드 (통계)
- 지점 / 재고 관리
- 예약 승인 / 관리
- 환율 관리
- 사용자 관리

## 요약

| 도메인 | 성격 | 비고 |
|---|---|---|
| 인증 | 진입점 | 작고 단순 |
| 환율 정보 조회 | 탐색형 | 예약 전 둘러보기 |
| 환전 예약 프로세스 | 트랜잭션 위저드 | 반드시 독립 |
| 예약 관리 | 사후 관리 | 프로세스와 분리 |
| 마이페이지 & 알림 | 부가 기능 | 통합 가능 |
| 관리자 | 별도 IA | 완전 분리 |

핵심은 **환전 예약 프로세스를 정보 조회와 묶지 않고 분리**하는 것이다. 위저드 흐름은 화면 수가 많고 단계별 이탈 지점을 따로 분석해야 하므로, 독립된 흐름도로 그려야 이후 UX 분석이 수월하다.
