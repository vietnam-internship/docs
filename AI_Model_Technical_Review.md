# AI 모델 기술 검토

## 1. 검토 목적

본 문서는 Tourist Currency Exchange Booking System의 AI 환전 시점 추천 기능에 적용할 수 있는 모델 후보를 비교하고, MVP 단계에서 사용할 기본 모델을 선정하기 위한 기술 검토 자료이다.

AI 기능은 서비스의 핵심 기능인 **실시간 환율 비교 및 온라인 예약 기능을 보조하는 역할**을 수행한다. 즉, AI는 환전 여부를 직접 결정하거나 투자 자문을 제공하는 기능이 아니라, 사용자가 환전 시점을 판단할 때 참고할 수 있는 보조 정보를 제공하는 것을 목적으로 한다.

## 2. 모델 비교

| 모델 | 장점 | 단점 | MVP 적용 판단 |
| --- | --- | --- | --- |
| 이동평균 기반 규칙 모델 | 구현이 쉽고 안정적이며 설명 가능성이 높음 | 데이터 패턴을 학습하는 AI 모델로 보기 어려움 | 폴백 로직으로 사용 |
| Linear Regression | 구현이 간단하고 회귀 계수를 통해 설명 가능함. 적은 데이터에서도 적용 가능 | 복잡한 비선형 패턴이나 장기 시계열 의존성을 반영하기 어려움 | MVP 기본 모델로 선정 |
| Ridge Regression | Linear Regression보다 과적합을 줄일 수 있음 | 모델 해석이 Linear Regression보다 약간 복잡함 | 데이터가 적거나 변동성이 클 경우 대안 모델 |
| Random Forest Regressor | 비선형 패턴 반영 가능, 성능이 비교적 안정적 | 설명 가능성이 낮고 모델이 무거워질 수 있음 | MVP에서는 제외, 향후 검토 |
| ARIMA | 전통적인 시계열 예측에 적합 | 파라미터 튜닝과 정상성 검토가 필요해 구현 부담이 있음 | 향후 고도화 |
| LSTM | 장기 시계열 패턴 학습 가능 | 많은 데이터와 학습 시간이 필요하고 설명 가능성이 낮음 | MVP 제외 |


## 3. 평가 항목별 모델 비교

| 항목 | Baseline Model | Linear Regression | LSTM/GRU | ARIMA |
| --- | --- | --- | --- | --- |
| 설명 가능성 | 높음. 임계값을 그대로 노출할 수 있음 | 높음. 회귀 계수를 저장하고 추천 근거로 설명 가능 | 낮음. 내부 판단 과정을 해석하기 어려운 블랙박스 모델에 가까움 | 중간. ARIMA 계수는 확인할 수 있으나, 회귀 모델보다 직관적인 해석은 어려움 |
| 과적합 리스크 | 없음. 학습 과정 자체가 없음 | 낮음. 필요 시 Ridge Regression으로 정규화 가능 | 높음. 데이터가 짧으면 과적합에 취약함 | 중간. 파라미터 선택과 정상성 처리에 민감함 |
| 재학습 비용 | 없음 | 낮음 | 높음. 데이터가 많아질 경우 GPU 또는 긴 학습 시간이 필요할 수 있음 | 중간. 주기적인 파라미터 재탐색과 검증이 필요함 |

위 비교를 기준으로 보면, MVP 단계에서는 **설명 가능성, 낮은 과적합 리스크, 낮은 재학습 비용**이 중요하다. 따라서 LSTM/GRU나 ARIMA보다 Linear Regression이 초기 적용에 더 적합하다. Baseline Model은 안정적인 폴백 로직으로 유지한다.

## 4. MVP 기본 모델 선정

MVP 단계에서는 **Linear Regression**을 기본 모델로 선정한다.

선정 이유는 다음과 같다.

1. **구현 난이도가 낮음**  
   Linear Regression은 데이터 전처리, 학습, 예측 파이프라인을 간단하게 구성할 수 있어 MVP 개발 일정에 적합하다.

2. **설명 가능성이 높음**  
   회귀 계수를 통해 어떤 피처가 예측 결과에 영향을 주었는지 비교적 쉽게 설명할 수 있다. 이는 사용자에게 AI 추천 근거를 제공해야 하는 본 서비스의 요구사항과 부합한다.

3. **적은 데이터에서도 적용 가능**  
   초기 서비스 단계에서는 충분한 장기 환율 데이터가 확보되지 않을 수 있다. Linear Regression은 LSTM과 같은 딥러닝 기반 모델보다 적은 데이터에서도 적용이 가능하다.

4. **MVP 목적에 적합함**  
   본 서비스에서 AI 기능은 핵심 거래 기능이 아니라 환전 시점 판단을 돕는 보조 기능이다. 따라서 MVP 단계에서는 복잡한 모델보다 안정적이고 설명 가능한 모델을 우선 적용하는 것이 적절하다.

## 5. 폴백 로직

Linear Regression 모델의 예측 결과를 사용할 수 없거나 데이터가 부족한 경우, **이동평균 기반 규칙 모델**을 폴백 로직으로 사용한다.

예시 기준은 다음과 같다.

- 현재 환율이 최근 N일 이동평균보다 낮고, `MA - 1σ` 이하인 경우: **환전 고려**
- 현재 환율이 최근 N일 이동평균보다 높고, `MA + 1σ` 이상인 경우: **대기 고려**
- 그 외 구간: **보통**

이를 통해 모델 예측이 불가능한 상황에서도 최소한의 추천 결과를 안정적으로 제공할 수 있다.

## 6. 향후 고도화 방향

MVP 이후에는 데이터가 충분히 축적된 뒤 다음 모델을 추가 검토할 수 있다.

- **Ridge Regression**: 데이터 수가 적거나 변동성이 큰 경우 Linear Regression의 과적합을 줄이기 위한 대안
- **Random Forest Regressor**: 비선형 패턴 반영이 필요한 경우 검토
- **ARIMA**: 전통적인 시계열 예측 성능 검토
- **LSTM/GRU**: 장기 시계열 패턴 학습이 필요하고 충분한 데이터가 확보된 경우 검토

## 8. 결론

MVP 단계에서는 **Linear Regression을 기본 AI 모델로 사용**하고, 데이터 부족 또는 예측 실패 상황에서는 **이동평균 기반 규칙 모델을 폴백 로직으로 사용**한다.

이러한 구조는 구현 부담을 줄이면서도 설명 가능성과 안정성을 확보할 수 있으며, AI 기능이 서비스의 보조 기능이라는 점을 명확히 보여준다.
